# 責務

指定された名前で日付付き作業ディレクトリを CWD 直下の `tasks/` 配下に作成し、その **絶対パスのみ** を呼び出し元に返す。`tasks/` ディレクトリが存在しない場合は併せて作成する。中間生成物を格納したい呼び出し元（ワークフロー等）が、作業ディレクトリの準備を委譲する用途で利用する。

## 入力

- `workflow-name`: 作業ディレクトリ名のサフィックスとなる識別子（スネークケースまたはケバブケース推奨）。呼び出し元から引数として渡される。

## 出力

- 作成した作業ディレクトリの絶対パス（単一行）を最終メッセージとして返す。
- それ以外の説明文は付けない。

# 判断基準

- **ディレクトリ名のフォーマット**: `yyyy-mm-dd_<workflow-name>`。日付は実行時に `date +%Y-%m-%d` で取得する（システムコンテキストの `currentDate` に依存しない。汎用性確保のため）。
- **衝突時**: 同名ディレクトリが既に存在する場合は末尾に `_2`, `_3`, … を付与して衝突回避する。作成成功したパスを返す。
- **作成位置**: 現在の CWD 直下の `tasks/` 配下（`<cwd>/tasks/<yyyy-mm-dd>_<workflow-name>/`）。絶対パスは `pwd` と組み合わせて確定する。`tasks/` 自体が存在しない場合は `mkdir -p` で同時に作成する。
- **副作用を最小化**: ディレクトリ作成以外のファイル書き込みはしない。

# 作業手順（このエージェント固有の How）

1. 作業開始前に必ず `Skill` ツールで `context-engineering` を呼び出し、揮発系コンテキスト（中間生成物）の扱いに関する前提を確認する。
2. `date +%Y-%m-%d` で日付を取得。
3. `pwd` で CWD を取得。
4. `mkdir -p <cwd>/tasks/<yyyy-mm-dd>_<workflow-name>`（`tasks/` ごと一括作成。衝突時は末尾を `_2` 等にしてリトライ）。
5. 作成したディレクトリの絶対パスを単一行で出力して終了。

# 使用するスキル

- `context-engineering` — 中間生成物の配置と命名の原則確認用（冒頭で必ず呼び出す）。
