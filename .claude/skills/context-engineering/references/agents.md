# エージェント

Claude Code に対する指示を起点に動作する実行主体。タスクのオーケストレーションを行うメインエージェントと、特定の役割を担うサブエージェントから構成される。

## サブエージェントの2類型

サブエージェントには次の2類型がある。本ドキュメント以降の議論で「サブエージェント」と書く場合は、特に断りのない限りユーザー定義サブエージェントを指す。

- **ユーザー定義サブエージェント** — 設計者が責務を定義して配置するサブエージェント。コーダー、レビュアー、デプロイ担当、プロジェクト管理者など。本設計の対象。
- **組み込みサブエージェント** — Claude Code 自体に標準搭載されているサブエージェント（Explore・Plan・General-purpose など）。Claude Code が必要に応じて自律的に spawn するため、本設計のオーケストレーションの外側で動く。設計者が spawn 判断を直接制御することは想定しない。

## メインエージェント

ユーザーからの指示を最初に受け取り、タスクを遂行する主体。必要に応じてサブエージェントを呼び出し、タスク全体のオーケストレーションを行う。

## ユーザー定義サブエージェント

メインエージェントから呼び出され、特定の役割を遂行する。独立したコンテキストウィンドウを持ち、他のエージェントとは分離された状態で作業する。

### 独立コンテキストの意義

独立コンテキストは、メインエージェントのコンテキスト圧迫を防ぐためのコンテキストエンジニアリング上の重要な手段であり、単なる分離以上の意味を持つ。

長時間タスクでメインエージェントが作業の詳細（読み込んだファイルの中身、試行錯誤の過程、使用したスキルの全文など）を抱え続けるとコンテキストが急速に圧迫される。スキルを使った具体的な作業をサブエージェントに委譲することで、作業の過程はサブエージェント内で完結し、メインエージェントには結果のみが返る。これによりメインエージェントは長時間にわたるオーケストレーションを継続できる。

### 定義の核となる要素

サブエージェントプロンプトに含めるべき要素。

- **責務** — その役割が何をすべきか
- **判断基準** — 迷った時にどう判断するか

エージェントは「What（何をするか）」と「Why（なぜそう判断するか）」の層であり、具体的な手法（How）は原則としてスキルに委ねる。How スキルへの切り出し基準（単一サブエージェント固有 How の扱いを含む）は `skills.md` の「判断基準（How スキルの切り出し）」を参照。

## エージェント間の関係

- ユーザー → メインエージェント：指示
- メインエージェント → サブエージェント：呼び出し（結果のみが返る）
- サブエージェント → 他のサブエージェント：原則として直接呼び出さない（オーケストレーションはメインエージェントに集約）
