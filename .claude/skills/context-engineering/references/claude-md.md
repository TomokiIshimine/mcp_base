# CLAUDE.md — プロジェクトの共通基盤

メインエージェント・サブエージェントの両方から常に参照される最上位のコンテキスト。常に薄く保つ。

## 位置づけ

ここに置いた情報は全エージェントのコンテキスト消費を恒常的に圧迫する。必要最小限の共通基盤のみを置き、役割固有・領域固有の情報は他の層に委ねる。

## 含めるべき内容（優先度順）

1. **プロジェクトの目的** — なぜこのプロジェクトが存在するのか
2. **開発における設計原則** — DRY、最小コンテキスト、疎結合など、プロジェクト全体を貫く技術的方針（詳細は対応ドキュメントへパス参照）
3. **技術スタック・規約の概要** — 詳細は各層に委ね、ここには簡潔な概要のみ
4. **ディレクトリ構成** — 主要な2階層までを記載し、プロジェクトの全体構造を把握できるようにする
5. **ドキュメントへのパス** — 各設計ドキュメントの所在を明示する

ドキュメント・ディレクトリへのパス記法は SKILL.md「ファイルパスの記法」を参照。

## 判断基準（情報の配置先）

情報の配置先を判定するときの CLAUDE.md 中心のルール。

- **全役割に共通して必要な情報** → CLAUDE.md に含める
- **特定の役割だけが必要とする情報** → サブエージェントプロンプト側に置く
- **複数の役割に関わるが全役割ではない情報** → CLAUDE.md には含めない
    - How に関わる内容であれば**スキル**に記載
    - それ以外であれば**ドキュメント**に記載

## CLAUDE.md の第二の役割：ナビゲーション

CLAUDE.md はディレクトリ構成とドキュメントパスを通じて、プロジェクト内のコードやドキュメントへのエントリーポイントとしても機能する。エージェントが全体像を素早く把握し、必要な場所へ辿り着けるようにする。

## アンチパターン（判断基準だけでは見落としやすい具体例）

- 特定のサブエージェントにしか関係しないコーディング規約を書く（→サブエージェントプロンプトへ）
- 全体の地図にもならない詳細設計を書く（→ドキュメントへ）
- How（具体的手法）を書く（→スキルへ）
