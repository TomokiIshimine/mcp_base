# infrastructure 層の規約

外部世界（DB・環境変数・ログ基盤）との結合点。usecase のポートを実装する具象
アダプタとして、入出力とマッピングに徹する。

## 責務

- usecase が定義したポートの具象実装を提供する。
- ビジネスルールを実装しない。判断ロジックは domain/usecase にあり、この層は I/O と
  変換に限定する。

## マッピング

- domain オブジェクト ⇄ 永続化表現（SQL の行・カラム）の変換をこの層に閉じ込める。
  永続化の形を上位層へ漏らさない。
- SQL リテラルはモジュール内の名前付き定数に集約する。テーブルごとにそのリポジトリ
  モジュール内へ同様に集約し、巨大な共通 SQL モジュールへ一極集中させない。

## 設定・シークレット

- 環境変数・接続設定・シークレットの読み取りはこの層に集約する。必須値の欠如は
  起動時に検出して落とす（fail-fast）。

## 例外

- 技術的な例外（PyMySQL 等）を捕捉し、ポートの契約上の例外（`RepositoryError` /
  `InvalidGreetingError` など）へ翻訳して送出する。生の技術例外を上位へ漏らさない。

## 依存

- `interface_adapter` に依存しない。

## ロギング

- 技術的な障害・回復（接続枯渇、待機からの回復、リトライ等）の記録はこの層が担う。
  レベルは調査用詳細=DEBUG、回復可能な異常=WARNING を基本とする。
- 上位へ例外を翻訳して送出する事象は、上位が失敗として記録するため、この層で重ねて
  ERROR を出さない。
