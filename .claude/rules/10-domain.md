# domain 層の規約

ビジネスの概念とルールを表現する最も内側の層。永続化・表示・framework の都合を
持ち込まない。

## 依存

- 依存先は標準ライブラリと domain 内のみ。`usecase`・`infrastructure`・
  `interface_adapter`・外部 framework（Streamlit, PyMySQL 等）を import しない。

## エンティティ・値オブジェクト

- 既定で `@dataclass(frozen=True)` の不変オブジェクトとして定義する。
- 不変条件（invariant）は `__post_init__` で自己検証し、不正な状態のインスタンスを
  生成させない。検証ロジックを上位層に肩代わりさせない。
- 永続化の都合（DB 識別子の有無・ORM 型・SQL のカラム構造）をエンティティの形に
  反映しない。

## ビジネスルール

- ドメインのルールはこの層に置く。usecase や interface_adapter にルールを染み出さ
  せない。

## 例外

- ドメイン固有の例外型のみを送出する。技術的な例外（DB・ネットワーク）をここで
  投げない。

## 副作用・ロギング

- I/O・時刻取得・乱数などの副作用を持たない。必要な値は引数で受け取る。
- この層ではログを記録しない（純粋・副作用なしを保つ）。
