# usecase 層の規約

アプリケーションのユースケース（処理の流れ）を組み立てる層。ビジネス不変条件は
domain に委ね、ここではオーケストレーションに徹する。

## 責務

- ユースケース単位の処理フローのオーケストレーションのみを担う。不変条件の判定は
  domain のエンティティ/値オブジェクトに委譲する。
- 副作用の括り（どこからどこまでを 1 ユースケースとするか）をこの層で決める。

## 依存性逆転

- 外部資源へのアクセスはポート（ABC）経由のみ。`infrastructure` の具象実装を
  import しない。具象は合成ルートから注入される前提で書く。
- ポート（リポジトリ等の抽象インターフェース）は usecase 側で定義し所有する。
  実装側の都合でポートの形を決めない。

## 境界の型

- 入出力は domain オブジェクトまたは DTO に限る。framework 型（Streamlit のオブ
  ジェクト・HTTP リクエスト等）をこの層で扱わない。

## 例外

- この層の契約上の業務例外（ポート境界の基底 `GreetingError` 系）を送出する。下位
  （infrastructure）の例外をそのまま漏らさず、ユースケースの語彙に翻訳する。

## ロギング

- 業務イベントの成功（正常系の完了）を INFO で記録するのはこの層の責務。
- 失敗の記録は上位（呼び出し側）に委ね、同じ失敗をこの層と上位で二重に記録しない。
