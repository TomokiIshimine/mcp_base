# 画面一覧

Streamlit が描画する画面（UI）の一覧と、各セクションの役割・入力項目・主要操作・対応機能。現行実装 `src/interface_adapter/auth_gate_view.py`（認証ゲート）と `src/interface_adapter/greeting_crud_view.py`（CRUD 画面）の実描画に基づく。

## このドキュメントの範囲

現行実態である Streamlit 製 greeting CRUD の画面を正として記述する。MCP ツール一覧ではない。本プロジェクトは将来 MCP サーバー化する構想があり、その際は本画面群が「MCP サーバーの管理画面」へ位置づけが移行する見込みだが、その具体仕様は本ドキュメントの対象外とする。名称と現行実態の関係・将来構想の詳細は `docs/01_architecture.md` の「プロジェクトの位置づけ」を参照。

機能仕様（入出力・前提・正常系の振る舞い）は `docs/02_features.md` を参照する。

## 認証ゲート

CRUD 画面は認証ゲート（`render_auth_gate`）の背後に置かれ、ゲートはログイン状態と認可判定に応じて次の 3 画面のいずれか 1 つだけを描画する。CRUD 画面が描画されるのは認可成立時のみで、未ログイン・認可不成立の画面では CRUD は一切描画しない。認可判定の業務ルール（管理者 Email との完全一致・検証フラグ）は `docs/02_features.md` を参照する。

| 分岐 | 条件 | 描画する画面 |
|---|---|---|
| 未ログイン | 未ログイン | ログイン画面（ログイン導線のみ） |
| 認可不成立 | ログイン済みかつ管理者として認可されない | アクセス拒否画面（拒否メッセージ＋再ログイン導線） |
| 認可成立 | ログイン済みかつ管理者として認可される | CRUD 画面（ログアウト導線を上部に添える） |

### ログイン画面（未ログイン）

- ページタイトル: `ログインが必要です`
- 表示: 情報メッセージ `この画面の利用には管理者 Google アカウントでのログインが必要です。`
- 主要操作: ボタン `Google でログイン`（押下で Google の OIDC ログインフローを開始する）

### アクセス拒否画面（認可不成立）

- ページタイトル: `アクセスが許可されていません`
- 表示: エラーメッセージ `このアカウントには管理者権限がありません。別のアカウントでログインし直してください。`
- 主要操作: ボタン `別のアカウントでログインし直す`（押下でログアウトし未認証状態へ戻す）

### ログアウト導線（認可成立時）

- CRUD 画面の上部にボタン `ログアウト` を添える。押下で現在のセッションをログアウトし未認証状態へ戻す。

## 画面構成

CRUD 画面は単一の Streamlit ページ（`render_crud`）で構成し、上から次の 4 セクションを縦に並べて描画する。

- ページタイトル: `Greetings CRUD`
- キャプション: `MySQL の greetings テーブルを操作します。`

| セクション | 見出し | 役割 | 対応機能 |
|---|---|---|---|
| 一覧 | 一覧 | 登録済み greeting を一覧表示する | 参照/一覧 |
| 新規作成 | 新規作成 | メッセージを入力し新規 greeting を登録する | 作成 |
| 更新 | 更新 | 既存 greeting のメッセージを書き換える | 更新 |
| 削除 | 削除 | 既存 greeting を削除する | 削除 |

更新・削除セクションは対象 id を要するため、一覧取得に成功しレコードが 1 件以上ある場合のみ描画する。一覧取得に失敗した場合、または 0 件の場合は新規作成セクションまでを描画する。

## 各セクションの詳細

### 一覧

- 入力項目: なし
- 主要操作: なし（ページ表示時に自動で取得・表示）
- 表示:
  - レコードあり: `id`・`message` の 2 列テーブル
  - レコードなし: 情報メッセージ `レコードがありません。下のフォームから作成してください。`
  - 取得失敗（利用者起因）: 警告メッセージ
  - 取得失敗（システム障害）: エラーメッセージ

### 新規作成

- 入力項目: テキスト入力 `メッセージ`
- 主要操作: ボタン `作成`
- 振る舞い: 送信成功で `作成しました。` を表示し、フォームをクリアして再描画する。失敗時は原因別メッセージ（利用者起因＝警告／システム障害＝エラー）を表示する。

### 更新

- 入力項目: 選択ボックス `更新する id`（一覧の id から選択）／テキスト入力 `新しいメッセージ`（選択 id の現在値を初期表示）
- 主要操作: ボタン `更新`
- 振る舞い: 送信成功で `更新しました。` を表示し再描画する。失敗時は原因別メッセージを表示する。

### 削除

- 入力項目: 選択ボックス `削除する id`（一覧の id から選択）
- 主要操作: ボタン `削除`
- 振る舞い: 削除成功で `削除しました。` を表示し再描画する。失敗時は原因別メッセージを表示する。

## 画面遷移

認証ゲートを通過後は単一ページ内で完結し、CRUD 操作による別ページへの遷移はない。作成・更新・削除の各操作は成功後に同一ページを再描画し、一覧へ結果を反映する。

認証に関わる遷移は次のとおり。ログイン画面で `Google でログイン` を押すと OIDC ログインフローへ遷移し、認可成立で CRUD 画面、認可不成立でアクセス拒否画面に到達する。CRUD 画面・アクセス拒否画面のログアウト導線を押すと未認証状態へ戻り、ログイン画面に遷移する。
