# 画面一覧

Streamlit が描画する画面（UI）の一覧と、各セクションの役割・入力項目・主要操作・対応機能。現行実装 `src/interface_adapter/greeting_crud_view.py` の実描画に基づく。

## このドキュメントの範囲

現行実態である Streamlit 製 greeting CRUD の画面を正として記述する。MCP ツール一覧ではない。本プロジェクトは将来 MCP サーバー化する構想があり、その際は本画面群が「MCP サーバーの管理画面」へ位置づけが移行する見込みだが、その具体仕様は本ドキュメントの対象外とする。名称と現行実態の関係・将来構想の詳細は `docs/01_architecture.md` の「プロジェクトの位置づけ」を参照。

機能仕様（入出力・前提・正常系の振る舞い）は `docs/02_features.md` を参照する。

## 画面構成

画面は単一の Streamlit ページ（`render_crud`）で構成し、上から次の 4 セクションを縦に並べて描画する。

- ページタイトル: `Greetings CRUD`
- キャプション: `MySQL の greetings テーブルを操作します。`

| セクション | 見出し | 役割 | 対応機能 |
|---|---|---|---|
| 一覧 | 一覧 | 登録済み greeting を一覧表示する | 参照/一覧 |
| 新規作成 | 新規作成 | メッセージを入力し新規 greeting を登録する | 作成 |
| 更新 | 更新 | 既存 greeting のメッセージを書き換える | 更新 |
| 削除 | 削除 | 既存 greeting を削除する | 削除 |

更新・削除セクションは対象 id を要するため、一覧取得に成功しレコードが 1 件以上ある場合のみ描画する。一覧取得に失敗した場合、または 0 件の場合は新規作成セクションまでを描画する。

## 各セクションの詳細

### 一覧

- 入力項目: なし
- 主要操作: なし（ページ表示時に自動で取得・表示）
- 表示:
  - レコードあり: `id`・`message` の 2 列テーブル
  - レコードなし: 情報メッセージ `レコードがありません。下のフォームから作成してください。`
  - 取得失敗（利用者起因）: 警告メッセージ
  - 取得失敗（システム障害）: エラーメッセージ

### 新規作成

- 入力項目: テキスト入力 `メッセージ`
- 主要操作: ボタン `作成`
- 振る舞い: 送信成功で `作成しました。` を表示し、フォームをクリアして再描画する。失敗時は原因別メッセージ（利用者起因＝警告／システム障害＝エラー）を表示する。

### 更新

- 入力項目: 選択ボックス `更新する id`（一覧の id から選択）／テキスト入力 `新しいメッセージ`（選択 id の現在値を初期表示）
- 主要操作: ボタン `更新`
- 振る舞い: 送信成功で `更新しました。` を表示し再描画する。失敗時は原因別メッセージを表示する。

### 削除

- 入力項目: 選択ボックス `削除する id`（一覧の id から選択）
- 主要操作: ボタン `削除`
- 振る舞い: 削除成功で `削除しました。` を表示し再描画する。失敗時は原因別メッセージを表示する。

## 画面遷移

単一ページ内で完結し、別ページへの遷移はない。作成・更新・削除の各操作は成功後に同一ページを再描画し、一覧へ結果を反映する。
